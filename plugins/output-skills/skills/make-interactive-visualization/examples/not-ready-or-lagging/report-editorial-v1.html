Why a node returns “Not Ready or Lagging” a minute after it rejoins

Three different code paths read one atomic boolean. The boolean refreshes every three seconds. Between two ticks, a follower can pass the readiness check, accept traffic, and fail it again—and external health-pollers and DNS caches keep the broken node in rotation for the rest of the minute.
